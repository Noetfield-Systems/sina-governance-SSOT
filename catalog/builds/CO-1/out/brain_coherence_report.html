Diagnostic findings

Stack Fragmentation

1 / 5 · fragmented

Fourteen tools were named across chat, three note apps, and two task trackers. The same customer record lives in at least four of them with no single source of truth.

Observed evidence:
  • tool inventory export
  • two conflicting customer lists (CRM vs spreadsheet)
Recommended first step: Name one source of truth per data type and route the rest to it.

Definition Drift

2 / 5 · gaps

The word 'active client' resolves three different ways depending on who is asked. The same question to the assistant returns different answers on different days.

Observed evidence:
  • three written definitions of 'active client'
  • two divergent answers to the same prompt
Recommended first step: Extract the current working definitions, have the founder ratify them, and lock them as the shared vocabulary.

Agent Theater

1 / 5 · fragmented

Two workflows are described as automated but a person re-checks and re-runs each one by hand. The automation reports success while the underlying step is redone manually.

Observed evidence:
  • screen recording of the manual re-run
  • success log with no linked output
Recommended first step: Retire the labels that claim automation until a step produces a checkable output on its own.

Missing Receipts

0 / 5 · absent

Decisions and automated runs leave no stored proof. When a result is questioned there is nothing to inspect after the fact.

Observed evidence:
  • no artifact directory found
  • outcomes described only in chat prose
Recommended first step: Have each run write a small stored record (inputs, output pointer, timestamp) so results can be checked later.

Dead Workflows

2 / 5 · gaps

Six documented workflows exist; three have not run this quarter and one points at a tool that was removed. Nobody owns the map, so stale steps are still followed by new hires.

Observed evidence:
  • workflow doc last-modified dates
  • one step referencing a decommissioned tool
Recommended first step: Mark each workflow live, stale, or retired, and delete the ones that no longer run.

Founder Overload

1 / 5 · fragmented

The founder is the routing layer for most decisions and the only person who can resolve a definition conflict. Progress stalls whenever the founder is unavailable.

Observed evidence:
  • calendar showing decision meetings back-to-back
  • queue that only the founder can clear
Recommended first step: Move the recurring decisions into written rules the team can apply without the founder in the loop.

Missing Decision-Spine

1 / 5 · fragmented

There is no single place that says how a decision is made, who approves it, and where it is recorded. Ten competent fragments exist but they do not share a spine.

Observed evidence:
  • no decision log
  • approvals happening in three separate channels
Recommended first step: Stand up one decision-spine document: the rule, the approver, and the record location for each recurring decision.
