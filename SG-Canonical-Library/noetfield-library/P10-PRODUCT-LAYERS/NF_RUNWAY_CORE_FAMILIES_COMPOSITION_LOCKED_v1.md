## Six core recipe families

Only these foundational families are recognized as cores:

```text
1. Research
2. Software Repair
3. Video
4. Data
5. Document
6. Web/Publish
```

| Family | Input → finished result | Current SG status |
|--------|-------------------------|-------------------|
| Research | Question/documents → cited decision-ready report | Initial Runway (parallel first-three) |
| Software Repair | Issue/failed CI → tested patch and green PR | Initial Runway (parallel first-three) |
| Video | Brief/assets → finished playable + downloadable video | Initial Runway (parallel first-three); Higgsfield = replaceable adapter |
| Data | CSV/XLSX/PDF/export → cleaned workbook, calcs, charts, anomalies, decision report | Next new family after `future_runway_gate` |
| Document | PDFs/forms/policies → structured extract, completed template, action list, document pack | After Data |
| Web/Publish | Audit/fix/publish intent → preview, patch, governed publication | Adapter-class; Circuit B PublisherAdapter path; later as full family |

Web/Publish must not become a generic landing-page generator ahead of Data and Document.

## Composition law (binding)

Higher-value products are compositions of cores — not new engines:

| Composite | Composition | Finished result |
|-----------|-------------|-----------------|
| Website Audit-to-Fix | Research + Software Repair + Web/Publish | Audit, prioritized defects, code patch, preview, green PR |
| Proposal/RFP | Research + Document | Compliance matrix, missing-evidence list, response draft, submission pack |
| Research-to-Deck | Research + Document renderer | Cited presentation, speaker notes, PDF/PPTX (output mode of Research Runway, not a seventh core) |
| Knowledge-to-SOP | Document + Research | SOPs, checklists, training guide, quizzes |
| Meeting-to-Execution | Document + tool adapters | Decisions, tasks, follow-ups, updated project records |
| Due Diligence | Research + Document + Data | Evidence room index, risk matrix, findings, decision memo |
| Content Repurposing | Video + Document + deterministic media tools | Clips, article, posts, captions, newsletter |

Composites remain backlog until their required cores pass acceptance. They are not parallel build SKUs in this cycle.

## Differentiation law (binding)

Generic chat-with-a-spreadsheet / draft-an-RFP / make-a-deck is not the product.

Each Runway Job must deliver a finished, reproducible decision package with:

- checks / verification
- traceable assumptions
- downloadable artifacts
- receipt / lineage to sources

Competitive note (non-authority): incumbent spreadsheet copilots, RFP platforms, deck generators, clip tools, site builders, and app agents already occupy generic chat-or-generate wedges. Noetfield sells the completed business-task package.
