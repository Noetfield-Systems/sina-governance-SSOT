# VERSIONING RULE (quick reference) — RATIFIED / CANONICAL

**Filename:** `{ARTIFACT_NAME}_v{major}.{minor}.{edit}_{YYYYMMDD-HHMM}.{ext}`
**In-file header (top):**
```
Version: vMAJOR.MINOR.EDIT
Created: YYYY-MM-DD (first save)
Last edited: YYYY-MM-DD HH:MM TZ
Edit log:
  v0.1.1 — initial draft
  v0.1.2 — <one line what changed>
```
- **major** — scope/phase change (Phase 1 → 2).
- **minor** — founder-approved substantive change.
- **edit** — every saved pass, even a typo. No silent re-saves.
- Timestamp = wall-clock local TZ. Superseded versions retained unless founder cleans up.

Full rule: `SSOT_VERSIONING_LAW_v0.1.1.md`. This supersedes the library's earlier looser footer form.
