# DOCTRINE — RECEIPTS, NOT DIAGRAMS (the funding thesis)

**Status:** Governing principle + funding thesis. Source: Cloud Kernel v1.3 §1, §13.
**First written:** 2026-07-03 12:34 PDT

## The law
**Fundable systems are proven by receipts, not diagrams.** Investors do not fund Neon, Cloudflare Workers, an 8-layer chart, or a PEVC drawing. They fund a factory that actually **ran**:
```
Input → Artifact → Evidence → Decision → Receipt
```
**One auditable vertical slice outweighs a hundred pages of architecture.**

## The One Truth Ticket (Kernel §13.1)
1. Pick one factory (video-ad or forge drain).
2. Run it end-to-end through SCAN→SHIP on the existing Railway FBE.
3. Emit receipts shaped exactly as artifact / evidence / decision (JSON now, SQL later).
4. Gate every advance on a validator/Hub receipt, **never agent self-report.**
5. Package the run as a one-screen investor demo.

## What NOT to build first (Kernel §13 — reviewers explicit)
✗ Neon migration ✗ CF Worker rewrite ✗ L8 OpenTelemetry ✗ model optimization — all premature.
**The whole kernel depends on the Execution Contract Validator — it comes first.**

## Relation
- Anti-theater + negative-proof: a receipt is proof-by-evidence; a diagram is a claim. Same law as "prove gates by rejection."
- ROC: the receipt is the reusable asset that proves a pattern ran — the thing you trade.
