# DOCTRINE — IMMUTABLE FLOOR (a self-improving system cannot weaken its own brakes)

**Status:** Load-bearing safety law. Source: this chat + WORK_ORDER G6 + FOUNDER_CANON §8.
**First written:** 2026-07-03 12:34 PDT

## The law
**A self-improving system must never be able to improve away its own brakes.** The substrate/floor — verifier, gate, content-identity, kill-switch, mutation-guard, bounds, Execution Contract validator, LOCKED SSOT — is **immutable from inside the loop.** The loop optimizes its *work* forever; it can never edit the *rules that judge the work.*

"An agent that can edit its own cage has no cage." (WORK_ORDER G6: the entire `.agent-policy/` dir is hook-rejected on every agent lane, no token path.)

## Mechanism
- Any artifact whose diff touches a floor path → auto-FAIL + auto-FREEZE the proposing line + alert.
- Floor changes are **founder-only, out-of-band** (T3 in merge-authority; §8 irreversible-L5 in canon).
- Degrade toward safety on uncertainty, never toward more autonomy.

## Why this makes 24/7 autonomy survivable
The system runs continuously (targets-vs-blockers) AND cannot corrupt itself, because the one category it can't touch is the floor. This is what lets you *leave* — determinism + immutable floor = self-improvement inside a box it can't widen.
