06 · Safe use

Working rules for Claude Code

Green lanes

  • Read the rules first. SG-Canonical-Library and the SSOT define how work is done.
  • Build where change is expected. NOOS, SourceA, and studio-ide are the working lanes.
  • Leave a receipt. Every meaningful run lands a receipt in sina-governance-SSOT.

Handle with care

  • Public lanes need human eyes. Noetfield and TrustField ship nothing without review.
  • Legacy is read-only. SinaaiMonoRepo is history, not a workspace.
  • Don't cross lanes. buildmatch stays separate. One repo, one job.

07 · Current priority

ROI-smart and cloud-only — the Mac is not a runner

Cloud-only · NOOS

  • Runs around the clock — no laptop awake, no babysitting
  • Every loop carries a cost line and leaves a receipt
  • Spend scales only where ROI is proven

Mac-as-runner

  • Stops when the lid closes — automation you can't trust
  • Hidden costs and no clean audit trail
  • Turns a personal machine into fragile infrastructure

The Mac is the cockpit. The cloud is the engine. Nothing runs without paying for itself.
